## 2단계 고민
* 결과에 대한 저장을 따로 하는 클래스가 필요할 듯 하다.
* 구조에 대해 좀 더 단순화할 필요가 있다.

## 2단계 진행 과정
* 일단 도메인 기반으로 개발해보자
* 도메인 개발 과정: 도메인 클래스생성 -> 테스트 생성 및 도메인 생성/삭제 테스트 -> 도메인 클래스 구현
* 공을 던지면 쓰러진 핀의 개수를 변경 -> FallenPins에서 처리
* 가장 적고, 핵심인 기능부터 구현(공을 던진다, Pin이 쓰러진다)
* 추가되는 기능은 항상 커밋을 먼저 달 것
